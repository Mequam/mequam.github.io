COMS-160

final projet write up

The most challenging thing for me in this assignment was getting the axWindowsMedia player actually added to the project. I spent many an hour going down rabbit hole after rabbit hole. Eventually (and after much internal conflict) I decided to use the official microsoft website for the media player api. normally I avoid their site like the plague as I find that reading through their mountain of information to get to the small mole hill that I need often takes far far too long. But alas, some problems are obscure enough that quick answers on stack overflow will not suffice (although ironically the answers I did find on the site pointed me towards the official documentation). So in conclusion the biggest take away that I learned from this project was how to read through technical documentation effectively.

Of course that was not the only thing that I learned along the way ; windows has a directory class, how to get the current working directory of the program, the windows media player does not like relative paths, the mp3u file format, extended mp3u file format, urls inside of mp3u files are an attack vector, the list goes on. Honestly there's so many little interwoven topics that I wouldn’t be able to remember all of them off the top of my head, nor would it make much sense to put them all here. But the most general skill that I got to take away from this was being able to read through technical microsoft information.
